Module 1 – Design Patterns and Principles
SOLID Principles
SRP (Single Responsibility Principle)

A class should have only one responsibility and one reason to change.

Example: Student class stores student data, StudentReport class generates reports.

OCP (Open Closed Principle)

Software entities should be open for extension but closed for modification.

Example: New shapes can be added without changing existing code.

LSP (Liskov Substitution Principle)

A derived class should be replaceable by its base class without affecting correctness.

Example: Sparrow can be used wherever Bird is expected.

ISP (Interface Segregation Principle)

Clients should not be forced to depend on interfaces they do not use.

Example: Separate Printer and Scanner interfaces.

DIP (Dependency Inversion Principle)

High-level modules should depend on abstractions, not concrete implementations.

Example: Notification class depends on IMessage interface.

Design Patterns

Design patterns are reusable solutions to common software design problems.

Creational Patterns
Singleton Pattern

Ensures that only one instance of a class exists.

Real-time Example: Logger, Database Connection.

Factory Pattern

Creates objects without exposing creation logic.

Real-time Example: Vehicle Factory.

Structural Patterns
Adapter Pattern

Allows incompatible interfaces to work together.

Real-time Example: Mobile Charger Adapter.

Decorator Pattern

Adds new functionality to an object dynamically.

Real-time Example: Coffee with extra toppings.

Proxy Pattern

Controls access to another object.

Real-time Example: Image Loading Proxy.

Behavioral Patterns
Observer Pattern

Defines one-to-many dependency between objects.

Real-time Example: YouTube Channel Notifications.

Strategy Pattern

Allows selecting algorithms at runtime.

Real-time Example: UPI, Credit Card, Net Banking Payments.

Command Pattern

Encapsulates a request as an object.

Real-time Example: TV Remote Control.

Advantages of Design Patterns
Improves code reusability
Reduces code duplication
Makes code maintainable
Enhances scalability
Follows industry best practices
Conclusion

Module 1 covers SOLID Principles and Design Patterns which help developers write clean, maintainable, reusable, and scalable software applications.
